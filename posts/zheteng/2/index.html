使用 Hugo 搭建一个和本站类似的博客

记录如何使用 Hugo + GitHub Pages 搭建一个适合个人使用的静态博客。

2026年7月8日星期三 阅读时长: 4 分钟
目录
  1. 前言
  2. 准备基础环境
  3. 开始搭建
  4. 上传到 GitHub Pages

前言

这篇文章想记录一下，如何搭建一个和本站类似的个人博客。

本站主要使用 Hugo + GitHub Pages 搭建。对个人博客来说，这套方案最大的优点是成本很低，可以做到 0 成本建站；同时，Hugo 生成的是静态网页，访问速度快，部署也比较稳定。

当然，它也不是完全没有门槛。和一些后台式博客系统不同，这种方式更新文章时，通常需要在本地修改 Markdown 文件，然后再把改动提交到 GitHub。对于完全不熟悉 Git 的人来说，前期会稍微麻烦一点。

另外，如果你不希望别人直接复制你的整个博客源码，也可以采用我现在使用的方案：

1
2

私有仓库：保存 Hugo 源码、文章 Markdown、主题配置
公开仓库：只保存 Hugo 生成后的网页文件

这样别人可以访问你的博客页面，但不容易直接拿到完整的 Hugo 项目源码。

准备基础环境

搭建 Hugo 博客本身并不算复杂。你可以直接搜索 Hugo 官网，然后查看官方的 Quick Start 文档。不过为了方便理解，我这里还是按照自己的实践过程重新整理一遍。

首先需要准备两个基础工具：

  • Hugo
  • Git

Hugo 下载地址：

Hugo 下载地址

Git 下载地址：

Git 下载地址

安装 Hugo 时，建议选择 Extended 版本。Extended 版本在普通 Hugo 的基础上，增加了对 SCSS、Sass、PostCSS 等资源处理能力的支持。很多现代 Hugo 主题都会用到 SCSS，比如本站使用的 Stack 主题，所以建议直接安装 Extended 版本。

安装完成后，可以使用下面的命令检查是否安装成功：

1
2

hugo version
git version

如果命令行无法显示 Hugo 版本，通常说明 Hugo 没有正确加入系统环境变量。

图中是我的 Hugo 安装路径。你实际配置时，需要替换成自己电脑上的路径。如果下载后的目录里没有 bin 文件夹，也可以自己新建一个 bin 文件夹，然后把 Hugo 可执行文件移动进去，再把这个 bin 路径加入系统环境变量。

开始搭建

基础环境准备完成后，可以先创建一个新的 Hugo 项目：

1
2
3

hugo new project quickstart
cd quickstart
git init

其中 quickstart 是项目名称，你也可以改成自己喜欢的名字。

Hugo 官方 Quick Start 示例里使用的是 Ananke 主题，但本站使用的是 Stack 主题。没有什么特别复杂的理由，主要官方展示的这个大佬的主题布局和视觉效果真的太好看了。

如果你也想使用 Stack 主题，建议一定安装 Hugo Extended 版本，否则主题中的 SCSS 资源可能无法正常处理。

在项目根目录执行：

1

git submodule add https://github.com/CaiJimmy/hugo-theme-stack/ themes/hugo-theme-stack

然后在 Hugo 配置文件中指定主题：

1

echo "theme = 'hugo-theme-stack'" >> hugo.toml

接着启动本地预览：

1

hugo server -D

如果命令正常运行，就可以在浏览器里打开本地地址，预览属于自己的博客了。

到这里，本地博客已经可以跑起来。但如果想让别人也能访问，就需要部署到 GitHub Pages。

上传到 GitHub Pages

首先需要创建一个 GitHub 仓库。仓库名称必须使用下面这种格式：

1

username.github.io

其中 username 替换成你的 GitHub 用户名。

为什么要这样命名？这是 GitHub Pages 的用户站点命名规则。如果感兴趣，可以查看官方文档：

GitHub Pages 快速开始

仓库可见性选择 Public。创建完成后，如果默认分支不是 main，可以在仓库设置中把默认分支调整为 main。

接着回到本地 Hugo 项目，在项目根目录执行：

1
2
3
4
5

hugo
git add .
git commit -m "Update Hugo site"
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main

这里简单解释一下：

  • hugo：生成静态网页文件
  • git add .：添加本地改动
  • git commit -m "Update Hugo site"：提交本次改动
  • git remote add origin ...：绑定远程 GitHub 仓库
  • git push -u origin main：把本地内容推送到 GitHub

推送完成后，可以进入 GitHub 仓库查看 Actions 或 Pages 的部署状态。

如果一切正常，访问下面这个地址就能看到你的博客：

1

https://你的用户名.github.io

到这里，一个基础的 Hugo + GitHub Pages 博客就搭建完成了。

后续更新文章

后续写文章时，可以在本地项目根目录执行：

1

hugo new posts/文章名字.md

这样会生成一篇新的 Markdown 文章。

我自己的文章目录会稍微再整理一下。比如我会把文章统一放在 content/posts 下面，然后按照分类或主题继续建立文件夹，每篇文章再单独放进一个独立目录。

这样做的好处是：文章和图片可以放在同一个文章目录下，后续引用图片会更方便。

例如文章目录可以这样组织：

1
2
3
4
5

content/posts/zheteng/2/
├── index.md
└── picture/
    ├── 2-cover.png
    └── 1.png

在文章中引用图片时，直接写：

1

![图片说明](picture/1.png)

写完文章后，执行下面几条命令即可更新博客：

1
2
3
4

git status
git add .
git commit -m "更新博客文章"
git push origin main

需要注意的是，Hugo 文章一般使用 Markdown 格式。如果之前没有接触过 Markdown，可以先学习一些基础语法，比如标题、换行、图片、链接、代码块等。

到这里，这篇 Hugo + GitHub Pages 建站教程就结束了。
